# ADR-MENU-002: 메뉴 상태 관리(Status) 및 기본 필터링 도입

## 상태
승인됨 (Accepted)

## 맥락 (Context)
- **이전 상태**: 메뉴의 존재 여부만 DB에 저장되며, 조회 시 삭제되지 않은 모든 메뉴가 노출됨.
- **문제점**: 재고가 없거나 임시 판매 중지된 메뉴를 관리할 수 없어, 불가능한 메뉴에 대한 주문이 들어오는 비즈니스 오류 발생 가능성이 높음.

## 결정 (Decision)
- 메뉴 엔티티에 `MenuStatus` (AVAILABLE, OUT_OF_STOCK, HIDDEN) 열거형 필드를 추가함.
- 일반 사용자 조회 API에서는 기본적으로 `AVAILABLE` 상태인 메뉴만 필터링하여 노출함.

## 근거 (Rationale)
- 비즈니스 운영의 유연성을 확보하고, 사용자에게 정확한 판매 정보만 제공하여 주문 경험의 신뢰도를 높이기 위함.
