# ADR-MENU-003: 전체 메뉴 목록 Redis 캐싱 전략

## 상태
승인됨 (Accepted)

## 맥락 (Context)
- **이전 상태**: 인기 메뉴만 캐싱되어 있으며, 일반 메뉴 목록 조회는 매번 RDB 쿼리를 수행함.
- **문제점**: 메뉴 목록 조회는 서비스에서 가장 빈번한 작업 중 하나로, 트래픽 증가 시 DB I/O 병목이 발생할 수 있음.

## 결정 (Decision)
- 전체 메뉴 목록 조회 결과(`ApiResponseDto<Page<MenuResponse>>`)를 Redis에 캐싱함.
- 메뉴 생성, 수정, 삭제 시 관련 캐시를 즉시 제거(Evict)하여 데이터 정합성을 유지함.

## 근거 (Rationale)
- DB 접근 횟수를 획기적으로 줄여 응답 속도를 극대화하고, RDB 리소스를 주문/결제와 같은 쓰기 작업에 더 집중시키기 위함.
