# ADR-POINT-005: 포인트 유효기간 관리 및 FIFO 소멸 전략

## 상태
승인됨 (Accepted)

## 맥락 (Context)
현재 포인트 시스템은 충전된 포인트의 만료 기한이 없어 법적 규제나 비즈니스 정책을 반영하기 어렵습니다. 또한, 여러 번 충전된 포인트 중 어떤 것을 먼저 사용할지에 대한 명확한 규칙이 필요하며, 사용자가 자신의 포인트 변동 내역을 상세히 조회할 수 있는 기능이 요구됩니다.

## 결정 (Decision)
1. **유효기간(`expiredAt`) 도입**:
    - `PointHistory` 엔티티에 `expiredAt` 필드를 추가하고, 충전 시 정책에 정해진 기간(예: 5년)을 자동 설정합니다.
2. **FIFO(First-In-First-Out) 차감 로직**:
    - 포인트 사용 시 만료 임박 포인트(충전 일자가 빠른 순)부터 우선적으로 차감하여 사용자의 불이익을 최소화하고 정합성을 유지합니다.
3. **포인트 이력 검색 API**:
    - QueryDSL을 활용하여 충전/사용/소멸 유형별, 기간별 필터링이 가능한 조회 기능을 제공합니다.

## 논의 과정 (Discussion)
*   **Developer**: "포인트를 사용할 때마다 매번 이력을 다 뒤져서 차례대로 깎으면 성능이 느려지지 않을까요?"
*   **Architect**: "사용자 한 명의 충전 이력은 수천 건을 넘기 어렵습니다. 인덱스가 걸린 `userId`와 `expiredAt`으로 조회하면 충분히 빠르게 처리 가능합니다."
*   **Developer**: "소멸된 포인트는 어떻게 처리하나요?"
*   **Architect**: "매일 자정 배치를 통해 `expiredAt`이 지난 포인트를 자동으로 차감 처리하고 `EXPIRED` 타입의 이력을 남기도록 설계합니다."

## 결과 (Consequences)
- **장점**: 실무 수준의 정교한 포인트 정책 구현. 사용자 신뢰도 향상.
- **단점**: 포인트 차감 로직의 복잡도 증가(단순 합계 차감 -> 개별 이력 추적).
