# ADR-003: 고성능 Kafka 메시지 수신 및 컨슈머 구성

## 상태
승인됨 (Accepted)

## 맥락 (Context)
Kafka 클러스터로 발행된 주문 내역을 데이터 수집 플랫폼으로 "실시간" 전송하기 위해, 애플리케이션 내에 효율적인 컨슈머 구현이 필요합니다. 파티션이 3개이므로 컨슈머 역시 이를 병렬로 처리할 수 있는 구조가 필요하며, 향후 확장성을 고려하여 다중 컨슈머 그룹을 지원해야 합니다.

## 결정 (Decision)
1.  **컨슈머 그룹 및 리스너 구성**:
    - `DataPlatformConsumer`: `data-platform-group` 소속으로 `coffee-orders` 토픽을 구독하여 외부 전송 로직(Mock API 호출 등)을 수행합니다.
    - `@KafkaListener` 어노테이션을 사용하여 스프링 부트의 선언적 리스너를 구축합니다.
2.  **병렬 처리 최적화**:
    - `concurrency` 설정을 통해 파티션 수에 맞춰 애플리케이션 내부 스레드를 3개로 설정하여 처리량을 극대화합니다.
3.  **오류 처리 및 재시도 전략**:
    - `DefaultErrorHandler`를 설정하여 일시적인 네트워크 오류 시 재시도 로직을 적용합니다.

## 결과 (Consequences)
- **장점**: 메시지 처리 지연(Latency) 감소 및 시스템 안정성 향상.
- **단점**: 컨슈머 스레드 관리에 따른 리소스 오버헤드 발생.
