# ADR-001: 분산 인프라 구성 (Kafka 클러스터 및 Redis)

## 상태
승인됨 (Accepted)

## 맥락 (Context)
다중 서버 및 다중 인스턴스 환경에서 고가용성과 확장성을 보장하기 위해 단일 브로커가 아닌 클러스터 구성의 Kafka와 분산 환경을 위한 Redis 인프라가 필요합니다. 특히 대규모 트래픽 처리를 위해 파티셔닝 전략과 다중 컨슈머 그룹 아키텍처를 도입하고자 합니다.

## 결정 (Decision)
1.  **Kafka 클러스터 구성**:
    - 고가용성 보장을 위해 3개의 브로커(Broker)로 구성된 Kafka 클러스터를 구축합니다.
    - KRaft 모드를 사용하여 별도의 Zookeeper 없이 클러스터를 관리합니다.
2.  **파티셔닝 및 복제 전략**:
    - 주요 토픽(`coffee-orders`)에 대해 최소 3개 이상의 파티션을 설정하여 병렬 처리 능력을 확보합니다.
    - `replication-factor`를 3으로 설정하여 브로커 장애 시에도 데이터 유실을 방지합니다.
3.  **Redis 구성**:
    - 분산 락(Redisson) 및 캐싱을 위한 단일 Redis 노드를 구성합니다 (향후 필요 시 Sentinel/Cluster 확장 고려).
4.  **Docker Compose 활용**:
    - 로컬 개발 및 테스트 환경의 일관성을 위해 모든 인프라를 `docker-compose.yml`로 관리합니다.

## 결과 (Consequences)
- **장점**: 브로커 장애에 대한 내결함성(Fault Tolerance) 확보, 컨슈머 확장을 통한 처리량 증대 가능.
- **단점**: 인프라 리소스 소모 증가 및 설정 복잡도 상승.
