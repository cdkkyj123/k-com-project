### 0. 문제 해결 전략 수립

> 해당 내용은 `README.md`에 필수로 첨부해야 합니다.
> 
- [ ]  설계 내용(ERD, API 명세서)
- [ ]  설계의 의도
- [ ]  선택한 문제해결 전략 및 이 선택을 위해 분석한 내용
- [ ]  기술적 선택 이유

### 1. 커피 메뉴 목록 조회 API

- [ ]  커피 정보(메뉴ID, 이름, 가격)을 조회하는 API를 작성합니다.

### 2. 포인트 충전 하기 API

- [ ]  결제는 포인트로만 가능하며, 포인트를 충전하는 API를 작성합니다.
- [ ]  사용자 식별값, 충전금액을 입력 받아 포인트를 충전합니다. (1원=1P)

### 3. 커피 주문/결제 하기 API

- [ ]  사용자 식별값, 메뉴 ID를 입력 받아 주문을 하고 결제를 진행합니다.
- [ ]  결제는 포인트로만 가능하며, 충전한 포인트에서 주문금액을 차감합니다.
- [ ]  주문 내역을 데이터 수집 플랫폼으로 **실시간** 전송하는 로직을 추가합니다.
(Mock API나 테스트 코드 등을 사용하여 사용자 식별값, 메뉴ID, 결제금액을 전송합니다.)

### 4. 인기 메뉴 목록 조회 API

- [ ]  최근 7일간 인기있는 메뉴 3개를 조회하는 API를 작성합니다.
- [ ]  메뉴별 주문 횟수가 정확해야 합니다.
